What is a Static Site Generator?

An SSG takes plain-text source files (Markdown, HTML templates, YAML config) and compiles them into a folder of pure HTML, CSS & JS — ready to be served from any web host.
No database. No PHP. No runtime.
Just files on a CDN — blazing fast and secure.
# Build pipeline
Source: Markdown + YAML
Templates: Liquid / Nunjucks
Output: static HTML/CSS/JS
Deploy: GitHub Pages / Netlify

A Brief History

  • 1991
    Tim Berners-Lee writes the first website — a static HTML file
  • 1995
    CGI scripts bring dynamic pages; static sites seen as "too simple"
  • 2008
    Jekyll released by Tom Preston-Werner (GitHub co-founder) — SSGs reborn
  • 2013
    Hugo launches with sub-millisecond build speeds
  • 2016
    JAMstack coined — JS, APIs, Markup paradigm takes hold
  • 2020s
    Next.js, Astro, SvelteKit blur line between SSG and framework

Jekyll Project Structure

my-site/
├── _config.yml   ← site-wide settings
├── _layouts/     ← HTML templates (default.html, page.html)
├── _includes/    ← reusable partials (header, footer)
├── _posts/       ← blog posts: YYYY-MM-DD-title.md
├── assets/       ← images, CSS, JS
├── index.md      ← home page
└── _site/        ← compiled output (auto-generated)

Deploying to GitHub Pages

Step 1 — Create a repo named username.github.io
Step 2 — Push Jekyll source to main branch
Step 3 — GitHub Actions auto-builds Jekyll
Step 4 — Site is live at https://username.github.io
Step 5 — Every git push triggers a redeploy
🆓 Free hosting · HTTPS included · Custom domains supported
Zero DevOps required — GitHub handles the entire build & deploy pipeline automatically.

CMS vs SSG — When to Choose?

Criterion Traditional CMS (WordPress) Static Site Generator
Speed Server renders per request Pre-built files, CDN-ready
Security Database attack surface No DB, minimal surface area
Hosting cost Server needed ($5–$50/mo) Free on GitHub Pages / Netlify
Non-dev editing GUI admin panel Markdown / headless CMS needed
Scalability Needs caching plugins Infinite scale via CDN
Best for E-commerce, large teams, portals Blogs, docs, portfolios, landing pages
